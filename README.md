## What This Is (And Isn't)

**What this is NOT:** a cloud service, a Python script, a web app, or a library requiring dependency installation. If you are looking for a source-code toolkit requiring `pip install` or `npm`, this repository will not serve that purpose.

**What it actually does:** PDF Toolkit 2026 is a standalone Windows executable designed for batch manipulation and processing of PDF documents. It provides a set of utilities for merging, splitting, compressing, extracting pages, and converting between limited image formats — all executed locally on your machine without sending data to any external server. The execution environment is a single `.exe` file with a graphical interface.

## System Prerequisites

Before downloading the executable, confirm that your environment meets the following requirements:

- **Operating System:** Windows 10 (build 1909 or newer) or Windows 11. 32-bit systems are **not** supported.
- **Architecture:** x64 (64-bit) processor.
- **Disk Space:** At least 250 MB of free space on the installation drive.
- **Memory:** 4 GB RAM minimum (8 GB recommended for large file batches).
- **Dependencies:** No external runtime is required. The executable bundles its own dependencies. The .NET Desktop Runtime 8.0 or higher is recommended, but the installer will attempt to auto-resolve this if missing.
- **Internet Connection:** Required only for the initial download. No phoning home after installation.
- **Permissions:** Administrator rights are required only during installation if the target directory is a protected system location (e.g., `Program Files`). Standard user rights are sufficient when extracted to a user-writable folder.

## What You Get

- **Batch Merge** 🔧 — Combine multiple PDFs into a single document with configurable page ordering.
- **Smart Split** 🔧 — Divide a PDF by page ranges, bookmarks, or file size thresholds.
- **Image Export** 🔧 — Extract pages as PNG or JPEG. Resolution and DPI are adjustable.
- **Compression Engine** 🔧 — Reduce file size using either lossless or lossy compression profiles. The default profile preserves vector quality.
- **Metadata Editor** 🔧 — View and modify document properties such as title, author, subject, and keywords. Does not corrupt digitally signed documents.
- **Page Reorder & Delete** 🔧 — Rearrange or remove pages silently. Uses a sandboxed page model before writing to disk.
- **OCR Pipeline (Experimental)** 🔧 — Applies Tesseract-based text layer insertion to scanned documents. Requires a one-time model download (~50 MB) on first use.

## Configuration & Usage

PDF Toolkit uses a straightforward graphical interface with a configuration profile system. Users may define batch profiles that save compression levels, output directories, and naming conventions in a JSON file located at `%APPDATA%\PDFToolkit\profiles.json`.

**Example Profile (JSON):**
```json
{
  "profileName": "ArchiveOptimized",
  "compressionLevel": 6,
  "outputFormat": "merged",
  "targetSizeMB": 10,
  "appendTimestamp": true,
  "outputPath": "C:\\Users\\Public\\Documents\\PDF_Output"
}
```

### CLI Usage (Advanced)

The executable also accepts command-line arguments for headless operation. The general syntax is:

```bash
pdf-toolkit.exe [operation] [input] [options]
```

| Argument        | Description                                      |
|-----------------|--------------------------------------------------|
| `--merge`       | Merge files listed in a text manifest.           |
| `--split`       | Split a single PDF by page range or file size.   |
| `--compress`    | Apply compression profile by name.               |
| `--extract-img` | Export pages as images.                          |
| `--output`      | Specify output directory or filename.            |
| `--profile`     | Load a saved profile from the profiles folder.   |

Example:
```bash
pdf-toolkit.exe --merge --input manifest.txt --output "C:\Output" --profile ArchiveOptimized
```
